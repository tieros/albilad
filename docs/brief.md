# Project Brief

## Objective

This project turns the Bank Albilad design work delivered by Commencis into a reusable operating model for both people and AI agents.

The main outcome is not only documentation. The real goal is repeatable execution:

- new designers should understand how the work was done
- AI agents should be able to continue the work without breaking continuity
- future design reviews and redesigns should follow the same standards

## Three Workstreams

### 1. Audit

The design system was reviewed to identify structural, foundational, and component-level issues.

Each audit item follows a fixed structure:

- Finding / Current Case
- Suggestion
- Explanation
- Severity

This work is design-system focused, not screen-focused.

### 2. Heuristic

Selected product screens and flows were reviewed from a UX perspective.

This work is screen-focused, not component-library focused.

It should remain consistent with the heuristic review structure already used in the Commencis report.

### 3. Boost

Selected screens were redesigned by combining:

- Audit insight
- Heuristic insight
- the original Albilad design system
- the Commencis Boost component kit
- the visual taste established in the delivered Boost screens

Boost is the synthesis layer. It is where system improvement and screen improvement meet.

## Relationship Between Workstreams

- Audit informs how components, foundations, patterns, and DS structure should improve.
- Heuristic informs how user journeys, clarity, feedback, recovery, and usability should improve.
- Boost uses both when redesigning real screens.

## Key Continuity Requirement

Future work must look and feel like a continuation of the delivered Commencis work.

That means:

- same writing logic
- same evaluation logic
- same prioritization logic
- same component thinking
- same visual taste direction in Boost outputs

## Non-Goals

This repository is not meant to:

- replace the original Figma files
- create a brand-new visual language
- produce detached one-off redesigns
- ignore the delivered audit and heuristic artifacts

## Success Criteria

This repository succeeds if:

- a new designer can understand the project without extra hand-holding
- an AI agent can route tasks correctly
- audit additions remain consistent with the existing report
- heuristic additions remain consistent with the existing report
- boost outputs remain consistent with the existing UI direction
