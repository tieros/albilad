# Audit Domain: Patterns

Use this domain for reusable UX patterns that sit above single components and below full screens.

## Inspect

- presence or absence of pattern documentation
- repeated combinations of components
- transactional flows
- empty states
- feedback patterns
- form patterns
- navigation patterns

## Common Failure Modes

- no pattern library at all
- patterns rebuilt from scratch in product work
- repeated UX structures without shared guidance
