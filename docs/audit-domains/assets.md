# Audit Domain: Assets

Use this domain for illustrations, imagery, icon assets, and asset organization.

## Inspect

- whether assets are modeled as a real library or treated as leftovers
- category clarity
- correct placement of non-component visual assets
- naming and findability
- consistency of visual asset treatment

## Common Failure Modes

- assets stored in vague catch-all groups
- no clear distinction between components and assets
- poor discovery and maintenance experience
