# Audit Domain: General Recommendations

Use this domain for cross-cutting design-system issues that are not limited to one component family.

## Typical Topics

- DS architecture
- source of truth problems
- categorization and naming
- library governance
- documentation quality
- branching and workflow
- developer handoff
- system-wide accessibility readiness

## Use This Domain When

- the issue appears in many components
- the issue is structural rather than local
- fixing it changes how the system is organized or operated

## Do Not Use This Domain When

- the issue belongs clearly to one component family
- the issue is only a screen-level UX problem
