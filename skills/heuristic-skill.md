# Heuristic Skill

## Purpose

Use this skill when the task is to evaluate an Albilad screen or flow through heuristic analysis.

## Required Reads

1. `design.md`
2. `docs/principles.md`
3. `docs/heuristic.md`
4. relevant existing heuristic examples from the delivered Figma file

## What This Skill Must Do

1. Identify the target screen or flow.
2. Check whether similar heuristic findings already exist.
3. Reuse the same evaluation logic and writing discipline.
4. Map new findings to the established heuristic model.
5. Produce output that can directly inform redesign work.

## Continuation Rules

- Keep the same analytical depth as the existing report.
- Avoid creating a brand-new evaluation framework.
- Preserve consistency in naming, grouping, and severity logic.

## When To Cross-Reference Audit

If a UX problem is clearly caused by a DS weakness:

- check whether the corresponding Audit issue already exists
- reuse that logic where helpful
- flag the DS dependency in the heuristic reasoning

## Output Contract

Return:

- screen or flow name
- heuristic principle
- concise finding
- user impact
- recommendation
- notes for redesign if relevant
